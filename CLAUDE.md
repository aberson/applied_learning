# applied_learning — Project Instructions

A personal applied-learning workspace. One self-contained folder per topic under `topics/`, each a hands-on ramp following the **70% hands-on / 20% mentoring / 10% coursework** method. Topic folders are generated by the global `/user-learn <topic>` skill. See [`plan.md`](plan.md).

## 1. Stack

| Layer | Tool |
|---|---|
| Language | Python 3.12 |
| Package mgmt | `uv` + `pyproject.toml` (virtual project; `package = false`) |
| Notebooks | `jupyter` + `nbconvert` (committed already-executed; graphics embedded) |
| Numerics / ML | `numpy`, `matplotlib`, `scikit-learn`, `torch` (CPU), `torchvision` |

## 2. Key commands

```bash
uv sync                                # install deps
uv run jupyter lab                     # interactive
bash scripts/run_all_notebooks.sh      # headless re-run of every notebook (runnability gate)
```

## 3. Layout

```text
applied_learning/
├── README.md  CLAUDE.md  plan.md  pyproject.toml  .gitignore
├── scripts/                  # run_all_notebooks.{sh,ps1}
└── topics/<slug>/
    ├── README.md
    ├── 10-coursework/        # essentials.md, papers-seminal.md, papers-frontier.md, related-topics.md, courses-resources.md
    ├── 20-mentoring/         # seed-ideas.md, open-questions.md, progress.md
    └── 70-handson/           # notebooks/  examples/  exercises.md
```

## 4. Conventions

- Each topic folder is self-contained and uses the numeric-prefix layout (`10-coursework/`, `20-mentoring/`, `70-handson/`) so the 70/20/10 ratio is visible at a glance.
- **Notebooks must run headless on CPU in seconds** — tiny nets, few epochs, dataset subsets. A notebook that won't execute clean doesn't ship.
- **No fabricated citations.** Cite author + year; only link an arXiv id you are certain of.
- **ASCII-only `print()`** in standalone `.py` examples (Windows cp1252 — see `../.claude/rules/windows-shell.md`).
- `20-mentoring/seed-ideas.md` applications are grounded in real workspace projects — read the target project's `CLAUDE.md` before proposing.

## 5. Second brain

This project is a `b2_project_goblin` target — `uv run goblin suggest applied_learning` (from the goblin repo) ranks next-learning-steps from the topic folders. The deliberate build-out hook is `b2_project_goblin/seeds/phase-g-learning-companion.md`; goblin's own source is not modified by this project.
